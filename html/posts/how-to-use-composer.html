Composer is a dependency manager for PHP. Composer will manage the dependencies you require on a project by project basis. This means that Composer will pull in all the required libraries, dependencies and manage them all in one place.

This kind of management for dependencies in a project is not a new concept, and in fact, much of Composer is actually inspired from npm from Node.js and Bundler from Ruby.

Why should you care?

If you’re working in Drupal 8, Drupal core uses Composer to manage core dependencies like Symfony components and Guzzle. Composer allows you to systematically manage a sprawling list of dependencies (and their subsidiary dependencies). It assists with locating, downloading, validating, and loading said packages, all while ensuring that exactly the right versions for each package are used. It can be a pain, but it's far better than using either nothing or developing a home-brewed solution.

Here’s some tips on how to start working with Composer:

System requirements

Composer requires PHP 5.3.2+ to run. A few sensitive php settings and compile flags are also required, but when using the installer you will be warned about any incompatibilities.

How to install composer

Linux and Mac

curl -sS [https://getcomposer.org/installer](https://getcomposer.org/installer) | php

How to find where composer is installed

`which composer`

This should return:

`/usr/local/bin/composer`

How to update composer:

`sudo /usr/local/bin/composer self-up`

How to install composer dependencies:

`composer install`

How to update composer dependencies:

`composer update`
